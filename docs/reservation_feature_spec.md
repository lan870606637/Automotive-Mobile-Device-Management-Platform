## 1. 功能概述

### 1.1 功能目的
为用户提供设备预约借用功能，允许用户提前预约未来某个时间段内的设备使用权，避免设备被他人占用。

### 1.2 适用设备类型
- 车机
- 手机
- 仪表
- 手机卡
- 其他设备

### 1.3 预约状态定义
| 状态 | 说明 |
|------|------|
| 待保管人确认 | 保管中设备预约，等待保管人确认 |
| 待借用人确认 | 借用中设备预约，等待当前借用人确认 |
| 待2人确认 | 借用中且有保管人的设备，需双方确认 |
| 已同意 | 预约已确认，等待预约时间到来 |
| 已拒绝 | 预约被拒绝 |
| 已取消 | 预约被取消 |
| 已过期 | 预约超时未确认 |
| 已转借用 | 预约已转为实际借用 |

## 2. 设备状态分类

### 2.1 按设备状态分类

#### 2.1.1 保管中设备（手机、手机卡、其他设备）
- **特征**: 有保管人，设备在保管人处
- **预约流程**: 需保管人确认
- **界面按钮**: "我要借用" + "预约借用"

#### 2.1.2 在库设备（车机、仪表）
- **特征**: 无借用人，设备在库房
- **预约流程**: 无需确认，直接成功
- **界面按钮**: "我要借用" + "预约借用"

#### 2.1.3 借用中设备（无保管人）
- **特征**: 车机、仪表被借用，无保管人
- **预约流程**: 时间重合需当前借用人确认
- **界面按钮**: "转给自己" + "预约借用"

#### 2.1.4 借用中设备（有保管人）
- **特征**: 手机等设备被借用，有保管人
- **预约流程**: 时间重合需双方确认
- **界面按钮**: "转给自己" + "预约借用"

## 3. 预约借用流程

### 3.1 通用预约流程

```
用户点击"预约借用"
    ↓
选择预约开始时间和借用天数
    ↓
系统检查冲突
    ↓
[无冲突] → 创建预约 → 显示Toast提示
    ↓
根据设备状态确定确认流程
    ↓
发送通知给相关人
```

### 3.2 Toast提示内容
```
已预约xxx设备，2026年2月27日 14:32:52-2026年2月28日 14:32:52
```

## 4. 各状态设备预约逻辑

### 4.1 保管中设备预约

#### 4.1.1 预约创建
1. 用户选择预约时间和天数
2. 系统检查设备是否可预约（未报废、损坏、丢失、封存、逾期）
3. 系统检查时间冲突
4. 创建预约，状态为"待保管人确认"
5. 通知保管人

#### 4.1.2 保管人确认流程
1. 保管人收到通知和弹窗
2. 保管人选择同意或拒绝
3. **同意**: 状态变为"已同意"
4. **拒绝**: 状态变为"已拒绝"，通知预约人
5. **弹窗关闭**: 可从通知中心重新打开

#### 4.1.3 预约到期自动转借用
1. 预约开始时间到达
2. 系统自动将预约转为借用
3. 通知当前借用人和保管人
4. 借用记录显示"自动借用 原因：预约时间已到"

#### 4.1.4 取消预约
- 待确认状态：预约人可取消
- 已同意状态：预约人可取消
- 取消后通知相关人

### 4.2 在库设备预约

#### 4.2.1 预约创建
1. 用户选择预约时间和天数
2. 系统检查时间冲突
3. 创建预约，状态直接为"已同意"
4. 无需通知任何人

#### 4.2.2 预约到期
1. 系统自动转为借用
2. 通知预约人

### 4.3 借用中设备（无保管人）预约

#### 4.3.1 时间重合判断
- **预约开始时间 < 当前借用预计归还时间**: 时间重合
- **预约开始时间 ≥ 当前借用预计归还时间**: 时间不重合

#### 4.3.2 时间重合处理
1. 创建预约，状态为"待借用人确认"
2. 通知当前借用人
3. 当前借用人弹窗确认
4. **同意**: 状态变为"已同意"
5. **拒绝**: 状态变为"已拒绝"

#### 4.3.3 时间不重合处理
1. 创建预约，状态直接为"已同意"
2. 无需确认

#### 4.3.4 特殊限制
- **逾期设备**: 不能预约，提示"该设备已逾期，需要原借用人归还后才能预约"
- **续期冲突**: 当前借用人续期时，检查是否与预约冲突

### 4.4 借用中设备（有保管人）预约

#### 4.4.1 时间重合处理
1. 创建预约，状态为"待2人确认"
2. 同时通知借用人和保管人
3. 任意一人确认后，状态变为"待另一人确认"
4. **必须双方同意**才能预约成功

#### 4.4.2 确认流程
```
待2人确认
    ↓
保管人同意 → 待借用人确认
    ↓
借用人同意 → 已同意
```

## 5. 预约冲突检测

### 5.1 创建预约时冲突检测

#### 5.1.1 与其他预约冲突
- 检测时间段是否重叠
- 排除自己的其他预约
- **冲突提示**: "当前时段已被预约，需要让xxx取消借用或更改预约借用时段"

#### 5.1.2 与当前借用冲突
- 借用中设备可以预约
- 时间重合需确认流程
- 时间不重合直接成功

### 5.2 借用时冲突检测

#### 5.2.1 与预约冲突
- 借用时间段与预约时间段重叠
- **冲突提示**: "该设备已被xxx预约，预约时间xxx-xxx，请修改借用时长或者联系xxx取消预约"

#### 5.2.2 自己预约自己借用
- 不视为冲突
- 预约时间到达后自动续期

### 5.3 转借时冲突检测

#### 5.3.1 强制转借
- 检测是否有预约冲突
- 显示强制转借弹窗
- 确认后取消相关预约

#### 5.3.2 预约取消通知
- 通知被强制取消的预约人
- 通知保管人
- 记录操作日志

### 5.4 续期时冲突检测

#### 5.4.1 检测逻辑
- 新归还日期与预约开始时间比较
- 如果新归还日期 > 预约开始时间，则冲突

#### 5.4.2 冲突提示
- **提示内容**: "无法续期，该设备已被xxx预约，预约时间xxx-xxx"
- **例外**: 预约人自己续期自己的预约，允许续期

## 6. 预约确认机制

### 6.1 弹窗确认

#### 6.1.1 弹窗内容
```
📅 预约确认

[预约人] 申请预约借用设备

设备：[设备名称]
预约时间：2026-02-28 09:00 至 2026-03-07 09:00

请确认是否同意该预约？

[拒绝] [同意]
查看设备详情 →
```

#### 6.1.2 弹窗行为
- 每次刷新页面自动弹出未确认的预约
- 弹窗可关闭，从通知中心重新打开
- 多个预约时，依次弹出

### 6.2 通知确认

#### 6.2.1 通知中心
- 显示待确认的预约通知
- 点击通知打开确认弹窗
- 通知包含设备信息和预约时间

### 6.3 超时处理

#### 6.3.1 待确认预约超时
- 预约开始时间到达仍未确认
- 自动标记为"已过期"
- 通知预约人预约已过期

#### 6.3.2 已同意预约超时
- 预约开始时间到达
- 自动转为借用
- 或设备不可用则自动取消

## 7. 预约到期处理

### 7.1 自动转借用

#### 7.1.1 转换条件
- 预约状态为"已同意"
- 预约开始时间 ≤ 当前时间
- 设备未被他人借用

#### 7.1.2 转换流程
1. 将预约状态改为"已转借用"
2. 更新设备借用信息
3. 创建借用记录
4. 发送通知

#### 7.1.3 自己预约自己借用
- 当前借用人 = 预约人
- 自动续期，更新预计归还时间
- 创建续期记录

### 7.2 设备被他人借用

#### 7.2.1 处理逻辑
1. 取消预约
2. 通知预约人"设备已被他人借用，无法转为借用"
3. 预约状态变为"已取消"

### 7.3 通知内容

#### 7.3.1 通知当前借用人
```
xxx设备预约时间已到，已自动转借给xxx
```

#### 7.3.2 通知保管人
```
xxx设备预约时间已到，已自动转借给xxx
```

#### 7.3.3 借用记录
```
自动借用 原因：预约时间已到
```

## 8. 特殊情况处理

### 8.1 设备状态变更

#### 8.1.1 设备损坏/丢失/报废
- 自动取消所有有效预约
- 通知预约人"设备已损坏/丢失/报废，预约自动取消"

#### 8.1.2 设备封存
- 自动取消所有有效预约
- 通知预约人"设备已封存，预约自动取消"

#### 8.1.3 设备删除
- **软删除**: 取消所有有效预约，通知预约人
- **物理删除**: 删除关联的预约记录

### 8.2 用户删除

#### 8.2.1 删除前检查
- 检查是否有借用设备
- 检查是否有保管设备
- 有则不允许删除

#### 8.2.2 预约处理
- 取消用户的所有未完成预约
- 通知相关保管人和借用人

### 8.3 强制转借

#### 8.3.1 冲突检测
- 检测是否有预约冲突
- 显示强制转借弹窗

#### 8.3.2 预约取消
- 取消已同意的未来预约
- 拒绝待确认的预约
- 通知所有被影响的预约人
- 通知保管人
- 记录操作日志

### 8.4 设备归还

#### 8.4.1 通知等待的预约人
- 归还时检查是否有等待的预约
- 通知第一个等待的预约人"设备已归还，可以借用了"

#### 8.4.2 通知保管人
- 告知保管人已通知等待的预约人

### 8.5 逾期设备

#### 8.5.1 预约限制
- 逾期设备不能预约
- 提示"该设备已逾期，需要原借用人归还后才能预约"

#### 8.5.2 续期限制
- 逾期超过3天不能续期
- 提示"无法续期，设备已逾期超过3天，请先归还后再借用"

## 9. 通知机制

### 9.1 通知类型

| 场景 | 通知对象 | 通知内容 |
|------|---------|---------|
| 预约创建 | 保管人/借用人 | 有人预约设备，请确认 |
| 预约同意 | 预约人 | 预约已同意 |
| 预约拒绝 | 预约人 | 预约被拒绝 |
| 预约取消 | 相关人 | 预约已取消 |
| 预约过期 | 预约人 | 预约已过期 |
| 转为借用 | 预约人/借用人/保管人 | 预约已转为借用 |
| 设备归还 | 等待的预约人 | 设备已归还 |
| 设备删除 | 预约人 | 设备已删除，预约取消 |
| 用户删除 | 相关人 | 用户已删除，预约取消 |
| 强制转借 | 预约人/保管人 | 设备被强制转借，预约取消 |

### 9.2 通知方式
- 弹窗通知（实时）
- 通知中心（可查看历史）
- 页面刷新后自动检查
- 邮件通知（定时发送）

### 9.3 邮件通知

#### 9.3.1 逾期3天邮件提醒
- **触发条件**: 设备逾期3天及以上
- **发送频率**: 每天上午9点
- **收件人**: 借用人
- **邮件主题**: 【设备管理系统】您有X个设备已逾期3天以上，请尽快归还
- **邮件内容**:
  - 逾期设备列表（名称、类型、逾期天数）
  - 系统登录链接
  - 提醒文案：请尽快登录系统归还设备，以免影响您的借用权限

#### 9.3.2 预约待确认邮件提醒
- **触发条件**: 有待确认的预约
- **发送频率**: 每小时
- **收件人**: 需要确认的人（保管人/借用人）
- **邮件主题**: 【设备管理系统】您借用/保管的xxx设备有人预约借用，请尽快确认
- **邮件内容**:
  - 设备名称、类型
  - 预约人姓名
  - 预约时间范围
  - 系统登录链接
  - 提醒文案：请在预约开始时间前登录系统确认是否同意该预约，逾期未确认将自动过期

## 10. 数据清理

### 10.1 自动清理

#### 10.1.1 清理条件
- 预约结束时间超过7天
- 预约状态为：已取消、已拒绝、已过期、已转借用

#### 10.1.2 清理频率
- 定时任务每分钟执行一次
- 清理时打印日志"已清理X条旧预约记录"

### 10.2 手动清理
- 用户可删除已拒绝、已取消、已过期的预约
- 只能删除自己的预约

## 12. 技术实现

### 12.1 定时任务
- **预约处理任务**: 每分钟执行一次
  - 处理到期的已同意预约
  - 处理过期的待确认预约
  - 清理旧预约记录
- **逾期提醒邮件任务**: 每天上午9点执行
  - 发送逾期3天及以上设备的提醒邮件
- **预约确认提醒邮件任务**: 每小时执行
  - 发送待确认预约的提醒邮件

### 12.2 数据库表
- **reservations表**: 存储预约信息
- **索引**: device_id + device_type, reserver_id, status

### 12.3 核心方法
- `create_reservation()`: 创建预约
- `check_reservation_conflict()`: 检查冲突
- `convert_reservation_to_borrow()`: 转为借用
- `cancel_reservations_by_device_status_change()`: 状态变更取消预约
- `process_reservations_schedule()`: 定时任务处理
