# 内网 DNS 服务器部署方案

## 方案概述

为了让公司所有人都能使用域名 `http://device.carbit.com.cn` 访问设备管理系统，需要搭建内网 DNS 服务器。

## 三种部署方案

### 方案 1：Windows DNS 服务器（推荐，适合有 Windows Server 的环境）

**适用场景：** 公司有 Windows Server 服务器

**部署步骤：**

1. 在 Windows Server 上，以管理员身份运行 PowerShell
2. 执行脚本：
   ```powershell
   .\install_dns_server.ps1
   ```
3. 输入设备管理系统的服务器 IP 地址
4. 脚本会自动安装 DNS 服务并配置域名解析

**客户端配置：**
- 在公司路由器/DHCP 服务器中，将 DNS 服务器地址改为安装 DNS 的服务器 IP
- 或在每台电脑的网卡设置中手动设置 DNS

---

### 方案 2：Docker DNS 服务器（推荐，最简单）

**适用场景：** 服务器上已安装 Docker

**部署步骤：**

1. 确保服务器已安装 Docker Desktop
2. 进入 `docker-dnsmasq` 目录
3. 双击运行 `start-dns.bat`
4. 脚本会自动获取本机 IP 并启动 DNS 服务

**客户端配置：**
- 将公司设备的 DNS 服务器设置为运行 Docker 的服务器 IP
- 管理界面：http://服务器IP:5380 （用户名：admin，密码：admin123）

---

### 方案 3：使用公司现有 DNS 服务器（最推荐，如果公司有）

**适用场景：** 公司已有内部 DNS 服务器

**操作步骤：**

联系公司 IT 部门，在内部 DNS 服务器上添加以下记录：

```
设备管理系统服务器 IP: 192.168.1.100 (请替换为实际 IP)

需要添加的 DNS 记录：
- device.carbit.com.cn       A记录    192.168.1.100
- admin.device.carbit.com.cn A记录    192.168.1.100
```

---

## 客户端 DNS 配置方法

### 方法 1：通过 DHCP 自动分配（推荐）

在公司路由器或 DHCP 服务器中，将 DNS 服务器地址改为内网 DNS 服务器 IP。

**优点：** 所有设备自动获取，无需手动配置

### 方法 2：手动配置（适合测试）

**Windows 系统：**
1. 打开"控制面板" → "网络和共享中心"
2. 点击当前网络连接 → "属性"
3. 选择"Internet 协议版本 4 (TCP/IPv4)" → "属性"
4. 选择"使用下面的 DNS 服务器地址"
5. 首选 DNS 服务器：输入内网 DNS 服务器 IP
6. 备用 DNS 服务器：8.8.8.8

**车机/安卓系统：**
1. 进入 WiFi 设置
2. 长按当前网络 → "修改网络"
3. 高级选项 → IP 设置改为"静态"
4. DNS 1：输入内网 DNS 服务器 IP
5. DNS 2：8.8.8.8

---

## 验证 DNS 是否生效

在命令提示符中执行：
```cmd
nslookup device.carbit.com.cn
```

如果返回正确的 IP 地址，说明 DNS 配置成功。

---

## 故障排查

### 问题 1：无法访问 DNS 服务器
- 检查防火墙是否开放 53 端口（TCP 和 UDP）
- 检查 DNS 服务是否正常运行

### 问题 2：能解析但无法访问网站
- 检查设备管理系统服务是否启动
- 检查防火墙是否开放 5000 和 5001 端口

### 问题 3：能访问内网但无法访问外网
- 检查 DNS 转发器配置是否正确
- 确保配置了上游 DNS 服务器（如 8.8.8.8）

---

## 端口说明

| 服务 | 端口 | 协议 | 说明 |
|------|------|------|------|
| 用户服务 | 5000 | TCP | 设备管理系统用户端 |
| 管理服务 | 5001 | TCP | 设备管理系统管理后台 |
| DNS 服务 | 53 | TCP/UDP | 域名解析服务 |
| DNS 管理 | 5380 | TCP | Docker DNS 管理界面（仅方案2） |

---

## 安全建议

1. **防火墙配置：** 只开放必要的端口（53, 5000, 5001）
2. **访问控制：** 限制 DNS 服务器的访问范围（仅内网）
3. **定期备份：** 备份 DNS 配置文件
4. **监控日志：** 定期检查 DNS 查询日志，发现异常访问
