# 设备转借通知逻辑说明

## 概述

本文档记录设备转借时的通知逻辑，包括通知对象、排除规则和通知内容。

## 相关代码文件

- `user_service/app.py` - 用户服务端转借接口
- `common/api_client.py` - 通知发送方法

## 通知对象

设备转借时，系统会通知以下对象（排除操作人自己）：

### 1. 原借用人

**代码位置：** `common/api_client.py:1458-1474`

**通知条件：**
- 存在原借用人 (`original_borrower` 不为空)
- 原借用人不是操作人自己 (`original_borrower != operator`)

**通知内容：**
```
您借用的设备「{device_name}」已被 {operator} 转借给 {new_borrower}。
```

### 2. 新借用人

**代码位置：** `common/api_client.py:1476-1489`

**通知条件：**
- 始终通知（只要用户存在）

**通知内容：**
```
{operator} 已将设备「{device_name}」转借给您。
```

### 3. 保管人

**代码位置：** `user_service/app.py:1808-1824`

**通知条件：**
- 设备有保管人 (`device.cabinet_number` 存在)
- 保管人不是原借用人 (`device.cabinet_number != original_borrower`)
- 保管人不是新借用人 (`device.cabinet_number != transfer_to`)
- 保管人不是操作人自己 (`device.cabinet_number != user['borrower_name']`)

**通知内容：**
```
您保管的设备「{device.name}」已被 {user['borrower_name']} 从 {original_borrower or '保管人'} 转借给 {transfer_to}
```

## 场景分析

| 场景 | 原借用人通知 | 新借用人通知 | 保管人通知 | 说明 |
|------|-------------|-------------|-----------|------|
| 保管人转借在库设备 | ❌ | ✅ | ❌ | 无原借用人；保管人是操作人自己 |
| 保管人转借他人借用的设备 | ✅ | ✅ | ❌ | 原借用人≠操作人；保管人是操作人自己 |
| 借用人转借给自己保管的设备 | ❌ | ✅ | ✅ | 原借用人=操作人；保管人≠操作人 |
| 借用人转借给第三方 | ❌ | ✅ | ✅ | 原借用人=操作人；保管人≠操作人 |

## 关键逻辑

### 原借用人参数处理

**代码位置：** `user_service/app.py:1804`

```python
api_client.notify_transfer(
    device_id=device.id,
    device_name=device.name,
    original_borrower=original_borrower or user['borrower_name'],  # 无原借用人时使用操作人
    new_borrower=transfer_to,
    operator=user['borrower_name']
)
```

当设备在库（无借用人）时，`original_borrower` 会被设置为操作人（保管人），这样在 `notify_transfer` 中由于 `original_borrower == operator`，不会发送通知给原借用人。
